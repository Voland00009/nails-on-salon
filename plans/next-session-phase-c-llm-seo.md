# Phase C — LLM discoverability (GEO)

## Зачем

Всё больше «найти мастера маникюра в National City» идёт через AI-ассистентов, не через Google search. Сайт нужно сделать **citable** — структурированно, машиночитаемо, с разметкой которая попадает в RAG retrieval контекст LLM.

## Стек экспертов для `/captain-debate`

- **Researcher (technical SEO + GEO specialist)** — landscape 2026: llms.txt adoption (Anthropic, OpenAI), AI Overviews ranking signals, FAQPage schema usage в SGE, robots.txt access for GPTBot/ClaudeBot/Google-Extended/PerplexityBot.
- **Architect (Schema.org / structured data engineer)** — обогащение `BeautySalon` до полного графа: Service objects per service (Manicure, Pedicure subclasses), FAQPage, Person для Tanya, OfferCatalog для прайса, Place + GeoCoordinates (уже есть). Совместимость с Google Knowledge Panel.
- **Critic (LLM RAG engineer)** — чтобы крошки контента, которые LLM лифтят дословно, не сломались: Q&A блоки с правильным framing, NAP-консистентность (Name/Address/Phone — phone здесь N/A, replace with Instagram handle as canonical), CTA в LLM ответах должно вести на Instagram DM.
- **Orchestrator** — синтез + приоритизация + concrete deliverables.

## Scope

1. **`public/llms.txt`** + **`public/llms-full.txt`** — манифесты для AI crawlers (high-signal pages list + structured content dump). Стандарт от Anthropic/Mintlify, поддерживается Claude/ChatGPT/Perplexity.
2. **`public/robots.txt`** — explicit allow для AI crawlers: GPTBot, ClaudeBot, Google-Extended, PerplexityBot, ChatGPT-User. Cloudflare по умолчанию может блокировать — проверить.
3. **`public/sitemap.xml`** — Astro sitemap integration (`@astrojs/sitemap`). Autogenerated.
4. **Schema.org enrichment** в `BaseLayout.astro` + `services.astro`:
   - `Service` object per service (с price, duration, includes)
   - `FAQPage` на /book
   - `Person` для Tanya (name, jobTitle, worksFor)
   - `OfferCatalog` собирающий все Service'ы
   - `accessibilityFeature` (от Phase B intersect)
5. **OG image** — сейчас в `BaseLayout.astro` TODO про `/og-image.jpg`. Сгенерировать proper 1200×630 hero.
6. **Twitter Card** — есть базовый, добавить `twitter:title`, `twitter:description`, `twitter:site`.
7. **NAP consistency** — Name (Nails On Salon), Address (single canonical string), "Phone" → Instagram handle `@nailsonsalon` как primary contact. Все вхождения проверить.
8. **Q&A блоки на /book** — переделать FAQ под FAQPage schema (mainEntity array of Question/Answer).
9. **Verify CDN не блокирует AI crawlers** — Cloudflare имеет AI-bot policies, по дефолту в 2026 может быть restrictive. Проверить в dashboard или через `curl -A "ClaudeBot/1.0" https://nails-on-salon.pages.dev`.

## Зависимость от Phase B

Phase B даёт Privacy/Terms — на них ссылается robots.txt и llms.txt. Поэтому Phase C **после** Phase B.
